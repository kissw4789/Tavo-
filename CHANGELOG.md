# 更新日志

## 0.88 - 2026-06-18

这是一次面向 TAV 0.88 的大版本更新。重点不是单个页面调整，而是把记忆系统、回忆系统、关系系统统一收口到新的消息级 API 方案下，并修复旧版本在世界书绑定、阶段指引、待处理事项排序和关系降级解释上的问题。

### 发布文件

- `Tavo-0.88.json`
- `releases/v0.88/插件包6-18适配Tavo0.88.json`

### 记忆系统

- 适配 TAV 0.88 消息级 API。
- 恢复记忆世界书自动绑定。
- 总开关关闭时会解绑记忆世界书，总开关开启时会重新绑定。
- 清空系统时会清理旧注入书绑定。
- 新增 0.88 升级提示弹窗。
- 新增旧数据迁移入口：
  - 迁移旧数据：本地转换，通常不消耗 API。
  - 从第一楼重新生成：重新扫描历史聊天，会消耗 API。
- 保留旧数据副本，迁移或重建后不直接删除旧数据。
- 优化长期记忆提示词，强化楼层、时间线、关系状态、待继承上下文。

### 待处理事项页

- 新增顶部筛选栏：
  - 待处理
  - 已完成
  - 全部
- 默认显示待处理事项。
- 标记完成后，事项进入已完成视图。
- 已完成事项不再因为 `updatedAt` 更新排到待处理区顶部。
- 全部视图先显示待处理，再显示已完成。
- 完成态卡片增加淡化和标题划线。
- 标记完成或重新打开后，当前卡片会自动收起。

### 回忆系统

- 帮助入口文案统一：
  - 未读：使用说明
  - 已读：帮助
- 打开首页、帮助和普通浏览不会自动消耗 API。
- 保留事件、印记、星图等旧能力。

### 关系系统

- 统一关系阶段为：
  - 陌路
  - 初识
  - 往来
  - 纠葛
  - 羁绊
  - 宿命
- 新阶段范围：
  - 陌路：0-4
  - 初识：5-14
  - 往来：15-39
  - 纠葛：40-64
  - 羁绊：65-89
  - 宿命：90-100
- 阶段由 `depth` 推导，避免旧存储阶段名污染。
- 不再在关系世界书里创建重复的 `rel_stage_*` 阶段条目。
- 运行时会清理旧阶段世界书条目。
- 旧阶段名只保留为识别和迁移映射，不作为新标准。
- 关系系统“使用说明”首次点击后会回退为“帮助”状态，并写入全局已读标记。

### 关系世界书与输出规则

- 重写 `关系系统输出规则`。
- 新版 `<RELATION>` 机器块格式：
  - `events`
  - `interactions`
  - `dynamicsResponse`
  - `milestone`
  - `unresolved`
- 移除旧模板中的 `pressureReports`，避免模型主动填旧字段。
- 新增 `interactions`，用于记录角色与角色之间的互动信号。
- 更新六阶段指引。
- 更新演绎总则。
- 更新阶段摘要。
- 更新成熟度提示。
- 阶段指引从“硬禁止”调整为“软边界”：AI 如实报告，系统负责降级或挂起。

### 世界书注入强度

- 关系世界书核心条目统一使用：
  - `injectionPosition:'atDepth'`
  - `injectionDepth:0`
  - `injectionRole:'system'`
- 修复 `injectionDepth:0` 被误处理成 `4` 的问题。
- 核心条目会强制更新，不会因为旧世界书里已有内容就跳过新版规则。

### 关系变化反馈

- 关系事件阶段不匹配时，系统会给出降级原因。
- UI 反馈会展示降级说明，避免只显示“降级”导致用户不理解。
- `interactions` 会写入最近角色互动和协议日志。
- `interactions` 当前不直接改用户与角色的关系值，避免角色间互动被错误串到账上。

### 平台兼容与错误修复

- `rawGet/rawSet` 增加旧存储 fallback。
- 清理旧报错文案 `platformCore.rawSet 不可用`。
- 保留对旧版本部分数据结构的读取兼容。
- 发布包结构保持稳定，正则脚本槽位未变化。

### 发布验证

- 提取目录检查通过。
- 发布 JSON 可解析。
- 深度验证通过。
- 原始包与发布包结构完全一致。
- `regex_scripts=14`，非空 `replaceString=12`。
- 空槽位保持 `[11, 12]`。
- 未发现 `新人必看` 文案残留。
- 未发现 `platformCore.rawSet 不可用` 文案残留。

### 已知边界

- `interactions` 当前是角色间互动记录，不是完整角色关系网结算。
- 关系系统仍偏内测；记忆系统是当前主稳定功能。
- 旧用户如果选择从第一楼重新生成，会消耗 API。
